How To Get Bugzilla

Instructions on how to obtain, install and configure Bugzilla are in the Installation and Maintenance Guide. That document also explains how to upgrade Bugzilla to a new version, move a Bugzilla installation between machines, and migrate from other bug-tracking systems.

If you are installing a new Bugzilla, the latest stable release series (and so the one you should choose unless you have a good reason for doing otherwise) is the 5.0 series. You will need this information when following the installation instructions.

If you are upgrading, note that you can upgrade directly from any old Bugzilla version to any new release—you do not have to go via the releases in between. You should upgrade to the latest stable series, 5.0, and read the Release Notes. If you want to know exactly what's changed from one version to the next, you can view the changelogs.

Bugzilla is licensed under the Mozilla Public License 2.0, and is thus both free and open source software.

Development Snapshot (5.1.1)

Bugzilla 5.1.1 is our latest development snapshot leading to Bugzilla 6.0.

Use of a development branch is at your own risk. They receive very little testing, so expect this release to be unstable. Users of 5.0 or older 5.1 releases are encouraged to try out new 5.1 releases in a testing environment and let us know of any problems that are found with the release.

A look at the new features that will be in Bugzilla 6.0 can be found in our latest Status Updates.
